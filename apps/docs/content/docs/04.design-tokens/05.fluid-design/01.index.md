## Overview

Fluid Responsive Design is a modern approach to creating layouts and typography that scale seamlessly across all screen sizes. Instead of jumping between fixed values at breakpoints, fluid design creates smooth, continuous transitions that adapt perfectly to any viewport width.

Styleframe's fluid design system is inspired by the [Utopia Fluid Type Scale](https://utopia.fyi/), which uses mathematical principles to calculate optimal sizes that maintain perfect proportions and readability across all devices.

## Why Fluid Design?

Traditional responsive design relies on breakpoints—discrete points where your design changes abruptly:

```/dev/null/traditional-responsive.css#L1-12
/* Traditional Responsive Approach */
.heading {
    font-size: 24px;
}

@media (min-width: 768px) {
    .heading {
        font-size: 32px; /* Sudden jump at breakpoint */
    }
}
```

This creates several problems:

- **Abrupt changes**: Text and spacing jump suddenly when crossing breakpoints
- **Limited coverage**: Only looks perfect at specific screen sizes
- **Complex maintenance**: Managing multiple breakpoints across numerous properties
- **Arbitrary decisions**: Choosing breakpoint values and sizes is often guesswork

**Fluid design solves these issues:**

```/dev/null/fluid-responsive.css#L1-4
/* Fluid Responsive Approach */
.heading {
    font-size: clamp(24px, 1.5rem + 2vw, 32px); /* Smooth scaling */
}
```

This single declaration:
- Scales smoothly from 24px to 32px
- Adapts to any viewport width between minimum and maximum
- Maintains perfect proportions at every size
- Requires no media queries

## How Fluid Design Works

Fluid design uses CSS `clamp()` functions combined with viewport-relative units to create smooth scaling:

```/dev/null/clamp-formula.css#L1-2
font-size: clamp(MIN, PREFERRED, MAX);
/* MIN: smallest size, PREFERRED: fluid calculation, MAX: largest size */
```

The fluid calculation uses a mathematical formula that considers:

1. **Minimum viewport width** (typically 320px for mobile)
2. **Maximum viewport width** (typically 1440px for desktop)
3. **Minimum value** (size at smallest viewport)
4. **Maximum value** (size at largest viewport)
5. **Current viewport width** (automatically calculated)

The result is a value that:
- Never goes below the minimum
- Never exceeds the maximum
- Scales proportionally in between
- Adapts in real-time as the viewport changes

## Benefits of Fluid Design

### Seamless Scaling

Text and spacing adapt continuously as the viewport changes. No more sudden jumps or awkward in-between sizes.

```/dev/null/example-seamless.ts#L1-15
import { styleframe } from 'styleframe';
import { useFluidFontSize, useScale, useScalePowers } from '@styleframe/theme';

const s = styleframe();

const { scaleMin, scaleMax } = useScale(s, { 
    min: 1.2,  // Mobile scale
    max: 1.25  // Desktop scale
});

const scaleMinPowers = useScalePowers(s, scaleMin);
const scaleMaxPowers = useScalePowers(s, scaleMax);

// Creates smooth scaling from mobile to desktop
const { fontSizeXl } = useFluidFontSize(s, { min: 16, max: 18 }, {
    xl: [scaleMinPowers[2], scaleMaxPowers[2]]
});
```

### Perfect Readability

Maintains optimal character count per line and visual hierarchy at every screen size, not just at breakpoints.

### Reduced Complexity

Replace dozens of media queries with a single fluid calculation:

**Before (Traditional):**
```/dev/null/before-traditional.css#L1-22
.heading {
    font-size: 20px;
}

@media (min-width: 480px) {
    .heading { font-size: 22px; }
}

@media (min-width: 768px) {
    .heading { font-size: 26px; }
}

@media (min-width: 1024px) {
    .heading { font-size: 30px; }
}

@media (min-width: 1440px) {
    .heading { font-size: 32px; }
}
```

**After (Fluid):**
```/dev/null/after-fluid.css#L1-4
.heading {
    font-size: var(--font-size--xl); /* Single fluid value */
}
```

### Mathematical Consistency

Uses proven modular scales (like the golden ratio or musical intervals) to ensure harmonious proportions:

```/dev/null/mathematical-scales.ts#L1-10
// Minor Third (1.2) - Subtle, elegant scaling
const { scaleMin } = useScale(s, { min: 1.2 });

// Major Third (1.25) - Balanced, versatile scaling  
const { scaleMax } = useScale(s, { max: 1.25 });

// Perfect Fourth (1.333) - Distinct, clear hierarchy
const { scale } = useScale(s, { default: 1.333 });

// Golden Ratio (1.618) - Dramatic, eye-catching scaling
const { scale } = useScale(s, { default: 1.618 });
```

### Performance Optimized

Modern CSS `clamp()` functions are:
- Calculated by the browser at rendering time
- More performant than JavaScript-based solutions
- Automatically optimized by the CSS engine

### Accessibility Friendly

Respects user preferences:
- Works with browser zoom
- Respects user font size settings
- Maintains proportional relationships
- Uses relative units (rem) for accessibility

## When to Use Fluid Design

Fluid design excels in these scenarios:

**✅ Perfect for:**
- Typography systems
- Spacing and padding
- Component sizing
- Container widths
- Any property that should scale smoothly

**⚠️ Use with caution for:**
- Complex layouts requiring precise alignment
- Grid systems with fixed column counts
- Properties where exact values matter (borders, shadows)

**❌ Not recommended for:**
- Binary states (visible/hidden)
- Layout breakpoints (still use media queries for these)
- Browser compatibility critical projects (requires modern browser support)

## Quick Start

Here's a complete example showing fluid typography in action:

::tabs
:::tabs-item{icon="i-lucide-code" label="Code"}

```/dev/null/quick-start.ts#L1-32
import { styleframe } from 'styleframe';
import { 
    useFluid, 
    useFluidFontSize, 
    useScale, 
    useScalePowers 
} from '@styleframe/theme';

const s = styleframe();
const { ref, selector } = s;

// Set up fluid breakpoints (320px - 1440px)
useFluid(s);

// Create scales for mobile and desktop
const { scaleMin, scaleMax } = useScale(s, { 
    min: 1.2,   // Minor Third for mobile
    max: 1.25   // Major Third for desktop
});

const scaleMinPowers = useScalePowers(s, scaleMin);
const scaleMaxPowers = useScalePowers(s, scaleMax);

// Generate fluid font sizes
const { fontSize, fontSizeLg, fontSizeXl } = useFluidFontSize(s, 
    { min: 16, max: 18 },
    {
        default: [scaleMinPowers[0], scaleMaxPowers[0]],
        lg: [scaleMinPowers[1], scaleMaxPowers[1]],
        xl: [scaleMinPowers[2], scaleMaxPowers[2]]
    }
);

// Use in your styles
selector('body', {
    fontSize: ref(fontSize) // Scales from 16px to 18px
});

selector('h1', {
    fontSize: ref(fontSizeXl) // Scales proportionally larger
});

selector('h2', {
    fontSize: ref(fontSizeLg) // Scales proportionally between
});

export default s;
```

## Available Composables

Styleframe provides three core composables for fluid design:

| Composable | Purpose | Use Case |
|------------|---------|----------|
| **`useFluid()`** | Set up fluid viewport ranges | Define min/max viewport widths |
| **`useFluidClamp()`** | Create fluid clamp calculations | Low-level fluid value generation |
| **`useFluidFontSize()`** | Generate fluid typography scales | Complete fluid type systems |

### `useFluid()`

Establishes the viewport range for fluid calculations. Call this once in your theme setup.

```/dev/null/use-fluid-example.ts#L1-11
import { styleframe } from 'styleframe';
import { useFluid } from '@styleframe/theme';

const s = styleframe();

// Use defaults (320px - 1440px)
useFluid(s);

// Or customize the range
useFluid(s, { 
    minWidth: 375,  // Start scaling at 375px
    maxWidth: 1920  // Stop scaling at 1920px
});
```

[Learn more about `useFluid()` →](/docs/design-tokens/fluid-design/clamp)

### `useFluidClamp()`

Creates individual fluid values using clamp calculations. Use this for custom fluid properties.

```/dev/null/use-fluid-clamp-example.ts#L1-14
import { styleframe } from 'styleframe';
import { useFluid, useFluidClamp } from '@styleframe/theme';

const s = styleframe();
const { variable } = s;

useFluid(s);

// Create a fluid spacing variable
const spacingLg = variable(
    'spacing--lg',
    useFluidClamp(s, [24, 48]) // Scales from 24px to 48px
);
```

[Learn more about `useFluidClamp()` →](/docs/design-tokens/fluid-design/clamp)

### `useFluidFontSize()`

Generates complete fluid typography scales with mathematical precision.

```/dev/null/use-fluid-font-size-example.ts#L1-20
import { styleframe } from 'styleframe';
import { useFluid, useFluidFontSize, useScale, useScalePowers } from '@styleframe/theme';

const s = styleframe();

useFluid(s);

const { scaleMin, scaleMax } = useScale(s, { min: 1.2, max: 1.25 });
const scaleMinPowers = useScalePowers(s, scaleMin);
const scaleMaxPowers = useScalePowers(s, scaleMax);

const { fontSize, fontSizeSm, fontSizeLg } = useFluidFontSize(s, 
    { min: 16, max: 18 },
    {
        sm: [scaleMinPowers[-1], scaleMaxPowers[-1]],
        default: [scaleMinPowers[0], scaleMaxPowers[0]],
        lg: [scaleMinPowers[1], scaleMaxPowers[1]]
    }
);
```

[Learn more about `useFluidFontSize()` →](/docs/design-tokens/fluid-design/typography)

## Browser Support

Fluid design requires modern CSS features:

- **CSS `clamp()`**: Chrome 79+, Firefox 75+, Safari 13.1+, Edge 79+
- **CSS `calc()`**: All modern browsers
- **CSS Custom Properties**: All modern browsers

For older browsers, consider providing fallback values:

```/dev/null/browser-fallbacks.css#L1-5
.heading {
    font-size: 24px; /* Fallback for older browsers */
    font-size: var(--font-size--xl); /* Fluid value for modern browsers */
}
```

## Best Practices

### 1. Start with Sensible Ranges

Choose viewport ranges that match your target devices:

```/dev/null/sensible-ranges.ts#L1-9
// Mobile-first (320px - 1440px)
useFluid(s, { minWidth: 320, maxWidth: 1440 });

// Tablet-first (768px - 1920px)
useFluid(s, { minWidth: 768, maxWidth: 1920 });

// Custom range
useFluid(s, { minWidth: 375, maxWidth: 1600 });
```

### 2. Use Consistent Scales

Stick to proven modular scales for harmonious results:

```/dev/null/consistent-scales.ts#L1-8
// Minor Third (1.2) - Subtle differences
const { scaleMin } = useScale(s, { min: 1.2 });

// Major Third (1.25) - Balanced hierarchy
const { scaleMax } = useScale(s, { max: 1.25 });

// Perfect Fourth (1.333) - Clear distinction
const { scale } = useScale(s, { default: 1.333 });
```

### 3. Test at Multiple Viewport Sizes

Fluid design looks different at every width. Test thoroughly:

- Minimum viewport (320px)
- Mid-range viewports (768px, 1024px)
- Maximum viewport (1440px)
- In-between sizes (500px, 900px, 1200px)

### 4. Combine with Media Queries When Needed

Fluid design doesn't replace breakpoints entirely. Use both:

```/dev/null/combine-approaches.ts#L1-22
import { styleframe } from 'styleframe';
import { useFluid, useFluidFontSize } from '@styleframe/theme';

const s = styleframe();
const { ref, selector, media } = s;

useFluid(s);
const { fontSize } = useFluidFontSize(s, { min: 16, max: 18 }, {
    default: [1, 1]
});

// Fluid typography
selector('.article', {
    fontSize: ref(fontSize)
});

// Layout breakpoint
media('(min-width: 768px)', ({ selector }) => {
    selector('.article', {
        maxWidth: '65ch'
    });
});
```

### 5. Document Your Decisions

Explain your scale choices and viewport ranges in your theme:

```/dev/null/document-decisions.ts#L1-17
import { styleframe } from 'styleframe';
import { useFluid, useScale } from '@styleframe/theme';

const s = styleframe();

// Viewport range: 320px (iPhone SE) to 1440px (standard laptop)
// This covers 95% of our user base according to analytics
useFluid(s, { minWidth: 320, maxWidth: 1440 });

// Scale ratios:
// - Minor Third (1.2) for mobile: subtle differences work better on small screens
// - Major Third (1.25) for desktop: more pronounced hierarchy on larger displays
const { scaleMin, scaleMax } = useScale(s, { 
    min: 1.2,   // Mobile scale
    max: 1.25   // Desktop scale
});
```

## Common Patterns

### Fluid Typography with Fixed Base

Keep body text at a fixed size, but scale headings fluidly:

```/dev/null/fixed-base-pattern.ts#L1-21
import { styleframe } from 'styleframe';
import { useFluid, useFluidFontSize, useScale, useScalePowers } from '@styleframe/theme';

const s = styleframe();
const { ref, selector } = s;

useFluid(s);

const { scaleMin, scaleMax } = useScale(s, { min: 1.2, max: 1.25 });
const scaleMinPowers = useScalePowers(s, scaleMin);
const scaleMaxPowers = useScalePowers(s, scaleMax);

const { fontSize, fontSizeLg, fontSizeXl } = useFluidFontSize(s, 
    { min: 16, max: 16 }, // Fixed base size
    {
        default: [scaleMinPowers[0], scaleMaxPowers[0]],
        lg: [scaleMinPowers[1], scaleMaxPowers[1]],
        xl: [scaleMinPowers[2], scaleMaxPowers[2]]
    }
);
```

### Fluid Spacing System

Apply fluid design to spacing for consistent density:

```/dev/null/fluid-spacing-pattern.ts#L1-22
import { styleframe } from 'styleframe';
import { useFluid, useFluidClamp } from '@styleframe/theme';

const s = styleframe();
const { variable, ref, selector } = s;

useFluid(s);

// Create fluid spacing variables
const spacingXs = variable('spacing--xs', useFluidClamp(s, [4, 6]));
const spacingSm = variable('spacing--sm', useFluidClamp(s, [8, 12]));
const spacing = variable('spacing', useFluidClamp(s, [16, 24]));
const spacingLg = variable('spacing--lg', useFluidClamp(s, [24, 36]));
const spacingXl = variable('spacing--xl', useFluidClamp(s, [32, 48]));

// Use in components
selector('.card', {
    padding: ref(spacing),
    gap: ref(spacingSm)
});
```
