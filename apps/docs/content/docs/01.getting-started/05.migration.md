## Overview

Already have a styling system and want to move to Styleframe? These guides map your current vocabulary to Styleframe's, side by side, so you can port a codebase incrementally instead of rewriting it.

Each migration guide gives you:

- A vocabulary table: your old class or token on the left, the Styleframe equivalent on the right.
- The config translation: how your theme values become Styleframe design tokens.
- Component patterns: how variant helpers (like CVA) become Styleframe recipes.
