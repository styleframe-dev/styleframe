Add `useToastRecipe` for transient, floating feedback notifications, modeled on the callout recipe. Following [Nuxt UI](https://ui.nuxt.com/docs/components/toast), the semantic `color` accent is scoped to the icon and progress bar — the surface stays a neutral card for every color — so the `useToastRecipe` surface has only `variant` (solid/soft/subtle), `size` (sm/md/lg), and `orientation` (horizontal/vertical) axes, and no compound variants. The `variant` axis carries the neutral surface treatment (solid = opaque neutral card with a hairline border, soft = faint borderless fill, subtle = faint fill with a hairline border), light/dark adaptive throughout, defaulting to `solid`. The `outline` style is dropped — a transparent-background surface is illegible when floating over arbitrary page content. The base departs from callout so it reads as a polished floating card: a soft, diffuse `@box-shadow.lg` elevation, well-rounded `@border-radius.lg` corners, and generous padding, sized to its content rather than full-width, and a `position: relative` / `overflow: hidden` container that anchors and clips the progress bar.

Adds the `toast-icon`, `toast-content`, `toast-title`, `toast-description`, `toast-progress`, and `toast-dismiss` sub-part recipes. `useToastIconRecipe` and `useToastProgressRecipe` each carry the `color` axis (9 colors) that paints the accent — both share a single `toastAccentColorVariants` map so they cannot drift, and both render with `currentColor` so an SVG or icon-font glyph picks up the accent automatically.

`useToastTitleRecipe` (semibold, snug) and `useToastDescriptionRecipe` (normal weight) establish the title/description type hierarchy via weight, both inheriting `currentColor` — now always the neutral body foreground — so the hierarchy holds on every variant. Both carry a matching `size` axis (sm/md/lg) on the literal font-size token — `sm` → `@font-size.sm`, `md` → `@font-size.md`, `lg` → `@font-size.lg`, default `md` — so title and description render at the same size at every step and the hierarchy is carried by weight alone.

`useToastProgressRecipe` renders a prominent countdown bar pinned flush to the toast's bottom edge and spanning the full width — `bottom`/`left`/`right: 0`, clipped to the toast's rounded corners by its `overflow: hidden` — that scales from full width to zero over a `--toast-duration` custom property (default `5s`). Pure CSS — the recipe owns the `toast-progress` keyframes, no runtime state toggling.
