# @styleframe/cli

The `styleframe` command-line interface, built on citty with consola output. Commands: `init`, `build`, `dtcg export|import`, `figma export|import`. This package has **no `bin` of its own** â€” [`src/index.ts`](./src/index.ts) default-exports a `run()` function, and the actual `styleframe` binary ships from the barrel package ([`engine/styleframe`](../../engine/styleframe/AGENTS.md), `dist/cli.js`), which calls it.

## Commands

- **`styleframe init [cwd]`** â€” creates `styleframe.config.ts` (skips if present), creates or merges `tsconfig.json`, adds dependencies to `package.json` from the baked-in `__INIT_VERSIONS__` map, then detects `vite.config.ts` / `nuxt.config.ts` and injects the plugin via magicast (warns with a docs URL if neither exists). The tsconfig `paths` entry for `virtual:styleframe` is only added when the project has `vue` without `nuxt` (`needsExplicitPaths` in [`src/commands/init.ts`](./src/commands/init.ts)) â€” Nuxt injects its own mapping via the plugin's `prepare:types` hook, and everything else resolves from the ambient shims. Merging is idempotent and never clobbers existing entries.
- **`styleframe build [entry]`** â€” loads the config with `@styleframe/loader` and writes build artifacts. `-o/--out` (default `styleframe`), `--clean`.
- **`styleframe dtcg export`** â€” spec-conformant DTCG. `-c` config, `-o` **output directory** (default `.`); writes `tokens.json` plus `tokens.resolver.json` when themes exist. AST â†’ DTCG in [`src/commands/dtcg/build-dtcg.ts`](./src/commands/dtcg/build-dtcg.ts); `TokenValue` â†’ primitive reduction in [`evaluate.ts`](./src/commands/dtcg/evaluate.ts). Irreducible CSS expressions and untypeable tokens are preserved via `dev.styleframe.*` extensions.
- **`styleframe dtcg import`** â€” DTCG JSON â†’ `*.styleframe.ts` code, generated by `@styleframe/figma`'s `generateStyleframeCode`. `-i` (required; file or directory), `-o` (default `tokens.styleframe.ts`), `--composables` (default true, uses `@styleframe/theme`), `--rem`, `--baseFontSize`, `--instanceName`.
- **`styleframe figma export|import`** â€” same shape as `dtcg` but in Figma's per-mode flavour: export writes one `<Mode>.tokens.json` per mode (reusing `build-dtcg.ts` + `flattenToFigmaDTCG`), import accepts a file or a directory of per-mode files.

Subcommands are lazy-loaded (`() => import("./commands/â€¦")`) to keep startup fast â€” keep new commands behind a dynamic import in [`src/index.ts`](./src/index.ts).

## Build-time defines

There is no generated `src/package.ts`. [`vite.config.ts`](./vite.config.ts) injects three compile-time constants: `__CLI_VERSION__` and `__CLI_DESCRIPTION__` (from this `package.json`), and `__INIT_VERSIONS__` â€” major-version ranges (`^N.0.0`) read from the workspace `package.json`s of `styleframe`, `@styleframe/{cli,core,loader,plugin,theme,transpiler,runtime}`. This is what `init` writes into user projects.

## Build & test

```bash
pnpm build            # tsc --noEmit + vite build (vite-plugin-node, ES + CJS)
pnpm dev              # vite build --watch
pnpm test             # vitest run --passWithNoTests
pnpm playground:init  # tsx playground/init.ts â€” scaffolds from playground templates
pnpm playground:build # tsx playground/build.ts
```

Tests are colocated in `src/` (currently `src/commands/dtcg/{build-dtcg,evaluate}.test.ts`). The command-level behavior (init/build against a real directory) is exercised via the playground scripts, not unit tests.

## Pitfalls

- **Version drift is baked in at build time.** `__INIT_VERSIONS__` snapshots workspace versions when the CLI is built â€” a stale CLI build writes stale dependency ranges into user projects. Rebuild after bumping package versions.
- **Users run the bin from the barrel.** Changing the CLI's public surface means the change reaches users through `styleframe` (the barrel package), which pins `@styleframe/cli` â€” check [`engine/styleframe/package.json`](../../engine/styleframe/package.json) wiring when renaming the default export or commands.
- **`dtcg export -o` is a directory, not a file path.** The filenames (`tokens.json`, `tokens.resolver.json`, `<Mode>.tokens.json`) are fixed.
- **`init`'s tsconfig handling must stay merge-only.** It parses JSONC, appends missing `include` patterns, and refuses to overwrite an existing `virtual:styleframe` paths mapping â€” regressions here clobber user config.
